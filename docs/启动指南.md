# mom_agent 启动指南

> 本文档覆盖本地从零到对话可用的完整启动流程,以及 **本地 vLLM** 与 **第三方 OpenAI 兼容 API**(DeepSeek / Moonshot / DashScope / OpenAI 等)两种推理后端的配置方法。

---

## 一、环境要求

| 组件 | 版本 | 说明 |
|---|---|---|
| Python | 3.10+ | 推荐 3.11 |
| Docker / Docker Compose | 任意现代版本 | 仅用于本地启 PostgreSQL + Redis |
| 推理后端(二选一或并存) | vLLM 部署的 Qwen3,或第三方 OpenAI 兼容 API | 本仓库 **不启动也不实现** 推理模型,只消费 |
| RAGFlow | 外部已部署 | 同上,知识库切片/索引/检索全部由 RAGFlow 负责 |

> 网络要求:本机需能访问 vLLM/第三方 API 端点、RAGFlow `POST /api/v1/retrieval`、本地 PG/Redis 端口。

---

## 二、一键启动(推荐)

启动脚本会自动:`docker compose up -d` → 检查 `.env`(没有则复制模板)→ `pip install` → `uvicorn` 起服务。

### Windows

```bat
start.bat
```

### Linux / macOS / Git Bash

```bash
./start.sh
```

> 首次运行若 `.env` 不存在,脚本会复制 `.env.example` 后 **退出**,请按下一节填好配置再重新运行。

启动成功后访问:**http://localhost:8000**

---

## 三、手动启动(脚本不适用时)

### 1. 起本地依赖(Postgres + Redis)

```bash
docker compose up -d
```

- PostgreSQL:`localhost:5432`,首次启动自动执行 `migrations/init.sql` 建表
- Redis:`localhost:6379`,短期记忆存储

健康检查:`docker compose ps`,二者状态均应为 `healthy`。

### 2. 准备配置

```bash
cp .env.example .env
```

按"四、推理模型配置"填写 LLM 与 RAGFlow 信息。

### 3. 安装 Python 依赖

```bash
pip install -r requirements.txt
```

### 4. 启动后端(自带前端静态文件,挂载在 `/`)

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

浏览器打开 **http://localhost:8000** 即可进入对话页。

## 四、推理模型配置

本项目支持两种推理后端,共用一套配置(`LLM_*` 优先,`VLLM_*` 作为兼容回退)。**两段并存**:不写 `LLM_*` 就走 `VLLM_*`,旧 `.env` 无需改动。

### 方案 A:本地 vLLM(默认)

```env
LLM_PROVIDER=vllm           # 会自动注入 Qwen3 的 enable_thinking 开关
VLLM_BASE_URL=http://你的vLLM地址:8001/v1
VLLM_API_KEY=EMPTY          # vLLM 通常不校验
VLLM_MODEL=Qwen3            # 与 vllm 启动时 --served-model-name 一致
VLLM_TEMPERATURE=0.3
VLLM_MAX_TOKENS=2048
```

### 方案 B:第三方 OpenAI 兼容 API

把 `LLM_*` 填上,vLLM 段可以留空。**`LLM_PROVIDER=openai` 时默认不注入 `enable_thinking`**(多数第三方不识别该参数)。

| 供应商 | `LLM_BASE_URL` | `LLM_MODEL` 示例 |
|---|---|---|
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` / `deepseek-reasoner` |
| Moonshot | `https://api.moonshot.cn/v1` | `moonshot-v1-32k` |
| 阿里 DashScope(Qwen) | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` / `qwen-max` |
| OpenAI | `https://api.openai.com/v1` | `gpt-4o-mini` |
| 智谱 BigModel | `https://open.bigmodel.cn/api/paas/v4` | `glm-4` |

示例(DeepSeek):

```env
LLM_PROVIDER=openai
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_API_KEY=sk-你的key
LLM_MODEL=deepseek-chat
LLM_TEMPERATURE=0.3
LLM_MAX_TOKENS=2048
# 高级:供应商专属参数(JSON)。例 OpenAI o 系列:{"reasoning_effort":"medium"}
LLM_EXTRA_BODY_JSON=
```

> 部分推理模型(DeepSeek-R1、Qwen3 思考模式)仍会以 `<think>…</think>` 输出思考链,前端会自动分流为可折叠的"思考过程",无需额外配置。

### RAGFlow(必填,否则检索不可用)

```env
RAGFLOW_BASE_URL=http://你的RAGFlow地址:9380
RAGFLOW_API_KEY=ragflow-你的key
RAGFLOW_DATASET_IDS=ds1,ds2          # 多个 dataset_id 逗号分隔
RAGFLOW_TOP_K=6                       # 召回片段数
RAGFLOW_SIMILARITY_THRESHOLD=0.2      # 相似度阈值
RAGFLOW_VECTOR_WEIGHT=0.3             # 向量权重 0~1
```

### 其它可选调优

```env
MAX_HISTORY_TOKENS=3000        # 超过即触发历史摘要压缩
KEEP_RECENT_MESSAGES=6         # 摘要后保留的最新消息条数
MODEL_CALL_RUN_LIMIT=4         # 单次 /chat 最多模型调用次数
MODEL_CALL_THREAD_LIMIT=200    # 单 session 累计上限
ENABLE_PROGRESS_EVENTS=true    # SSE 进度事件
ENABLE_MODEL_RETRY=true        # 瞬时错误指数退避重试
ENABLE_PII_REDACTION=true      # 输入 PII 脱敏(默认仅 email/credit_card)
SKILLS_DIR=./skills            # Skill 加载目录
```

## 五、验证启动

启动成功后:

1. 浏览器打开 `http://localhost:8000`,看到对话界面。
2. 发送一条消息(任意),观察:
   - 顶部出现 `meta` 事件(session_id 等)
   - 若开启思考模式,先有可折叠的"思考过程"
   - 然后流式打出答复,末尾若引用知识库会带 `[来源N]`
3. 命令行 uvicorn 日志能看到 `POST /chat 200`。

## 八、常见问题

**Q1:`docker compose up -d` 失败?**
确认 Docker Desktop 已启动;Windows 上若提示端口占用,改 `docker-compose.yml` 的 `5432`/`6379` 映射端口。

**Q2:启动报 `pydantic_core._pydantic_core.ValidationError: VLLM_BASE_URL Field required`?**
`.env` 未填或未被 uvicorn 进程读取。`VLLM_BASE_URL` 与 `VLLM_MODEL` 即使走第三方 API 也必须填(可填占位值,如 `http://localhost:9/v1` / `placeholder`),因为它们是 `LLM_*` 留空时的 **回退默认值**,字段本身仍是必填。

**Q3:换成第三方 API 后报 `Unrecognized request argument: chat_template_kwargs`?**
说明 provider 选错。把 `LLM_PROVIDER=openai`(而非 `vllm`),或显式设 `LLM_ENABLE_THINKING_PARAM=false`,系统就不会再注入该参数。

**Q4:RAGFlow 检索没结果?**
- `RAGFLOW_DATASET_IDS` 必须是 RAGFlow 中真实存在的 dataset_id
- 该数据集需已上传并完成索引
- `RAGFLOW_BASE_URL` 在 uvicorn 进程内可达(注意不要写成容器内地址)
- 检索失败会被静默降级为空,看 uvicorn 日志的 `ragflow` 警告

**Q5:深度思考开关无效?**
- vLLM/Qwen3:确认 vLLM 版本支持 chat template 中的 `enable_thinking` 变量
- 第三方 API:多数不支持运行时切换,需要选带"推理"后缀的模型(如 `deepseek-reasoner`、OpenAI o 系列)并通过 `LLM_EXTRA_BODY_JSON` 传供应商专属参数

**Q6:如何看更详细的日志?**

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --log-level debug
```

去掉 `--reload` 还可减少重启带来的噪音。

**Q7:同一 session 报 429?**
`app/api/chat.py` 对同一 `session_id` 做了串行化,前端不要并发同一会话。换 `session_id` 或等上一条结束。

**Q8:改了 `.env` 不生效?**
配置经 `@lru_cache` 缓存,**必须重启 uvicorn 进程** 才会重新读取。
