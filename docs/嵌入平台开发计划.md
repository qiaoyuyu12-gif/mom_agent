# 把 mom_agent 嵌入 workbench 平台 —— 开发计划

> 目标平台:`http://172.25.2.135:8080/workbench`
> 嵌入形态:右下角悬浮按钮 + 点击展开的对话面板
> mom_agent 部署位置:与平台同主机不同端口(8000)

---

## 1. 背景与目标

外部 workbench 平台需要嵌入 mom_agent 作为知识库助手。需求:

- 平台页面右下角出现一个圆形悬浮按钮(类似在线客服)。
- 点击按钮:展开一个对话面板,面板内是 mom_agent 完整 UI(深度思考、Skill、历史会话、来源标注全部保留)。
- 平台已登录的用户身份(user_id)需传给 mom_agent,使得**会话归属于该平台用户**——同一用户无论是在 workbench 浮窗里聊,还是直接访问 mom_agent,看到的历史会话一致。
- 关掉浮窗再打开,对话状态保留(iframe 不销毁)。

mom_agent 现状对嵌入很友好:

- 后端 `app/main.py:35-40` 已配置 `CORSMiddleware(allow_origins=["*"])`,跨端口请求与 SSE 不会被拦截。
- 前端 `web/index.html` 是单文件、纯 inline 样式与脚本、无 CDN 依赖,iframe 加载零外部依赖。
- `/chat`(`app/api/chat.py:62`,字段 `user_id: str | None`)与 `/sessions?user_id=...`(`app/api/chat.py:150`)已支持外部传入用户 ID。**后端零改动可对接。**

---

## 2. 总体方案

**iframe 嵌入 + 平台侧注入一段 widget loader JS。**

| 角色 | 工作 |
| --- | --- |
| mom_agent 前端 | 识别 URL query `?embedded=1&user_id=...&session_id=...`;嵌入模式下优先用平台传入的 user_id;CSS 自适应 360–440px 小宽度容器 |
| 新增 `web/embed.js` | 通用 widget loader,平台粘一段 `<script>` 即可获得悬浮按钮 + 弹出面板 + iframe 容器 |
| mom_agent 后端 | **不改动**(CORS 已开,user_id 已是现成参数) |
| workbench 平台 | 在页面模板尾部加 2 行 `<script>`,把当前登录用户 ID 传给 `MomoAgent.init()` |

不选其它方案的原因:Web Component / Shadow DOM 注入式 widget 要把整个 mom 前端搬出去重构,改动量大;浏览器扩展方案需每用户安装,运维成本高。**iframe 是隔离性、改动量、运维成本三者平衡最好的方案。**

## 3. 改动清单

### 3.1 `web/index.html` —— 嵌入模式自适应(主要工作)

在脚本顶部新增 URL 参数解析:

```js
const params = new URLSearchParams(location.search);
const IS_EMBEDDED   = params.get("embedded") === "1";
const EMBED_USER_ID = (params.get("user_id") || "").trim();
const EMBED_SESSION_ID = (params.get("session_id") || "").trim();
if (IS_EMBEDDED) document.body.classList.add("embedded");
```

**user_id 决策**(改 `web/index.html:311-316`):

- `IS_EMBEDDED && EMBED_USER_ID`:直接用平台传入的 user_id,不读/写 `localStorage["mom_user_id"]`(避免污染独立访问时的匿名身份)。
- 否则:保留现有 localStorage 逻辑。

**session_id 决策**(改 `web/index.html:304-308`):

- `IS_EMBEDDED && EMBED_SESSION_ID`:优先采用,支持平台后续做"从某记录直接打开对话"的深链。
- 否则:保留现有 `localStorage` / `genUuid` 逻辑。

**CSS 自适应**(在 `<style>` 末尾追加 `body.embedded { ... }`):

| 元素 | 嵌入模式下的调整 |
| --- | --- |
| `header` | 压缩为单行,隐藏 sid 显示 (`#sid` 那段) 以减少噪声 |
| `#history-drawer` | 宽度从 `min(440px, 92vw)` 改为 `100%`,iframe 内整窗滑出更合理 |
| `.msg` | `max-width` 从 `78%` 提高到 `88%`,小宽度下气泡更舒展 |

**postMessage 支持(v2,先不做)**:监听 `window.message` 事件接收父窗口的 `{type:'momo:set-user', user_id}` 以支持平台用户切换登录后实时改身份。第一版只做 URL query 即可。

### 3.2 新增 `web/embed.js` —— 平台侧 widget loader

纯原生 JS,无依赖。导出全局 `window.MomoAgent.init(opts)`。**所有 CSS class 加 `momo-` 前缀**避免与平台样式冲突。

平台接入方式(目标用法):

```html
<script src="http://172.25.2.135:8000/embed.js"></script>
<script>
  MomoAgent.init({
    origin: 'http://172.25.2.135:8000',
    userId: '<平台模板渲染的当前用户ID>',
  });
</script>
```

API 设计:

```js
window.MomoAgent = {
  init({ origin,             // 必填:mom_agent 服务地址
         userId,             // 必填:平台当前用户 ID
         sessionId = null,   // 可选:深链场景指定打开某个会话
         position = 'bottom-right',
         width  = 400,
         height = 640,
         title  = '知识库助手' }) {
    // 1. 注入 <style>(浮按钮 + 面板 + 遮罩 + 动画)
    // 2. 创建悬浮按钮(圆形,带 SVG 图标,fixed 右下)
    // 3. 创建面板容器(默认隐藏,fixed 定位,头部 title + 关闭按钮 + iframe)
    // 4. 点击按钮:toggle 面板;惰性创建 iframe(src 拼 ?embedded=1&user_id=...&session_id=...)
    // 5. 关闭按钮:仅隐藏面板,iframe 保留以延续会话状态(只在 destroy 时移除)
  },
  open()  { /* 程序化打开 */ },
  close() { /* 程序化关闭 */ },
  destroy() { /* 移除全部 DOM 与 iframe */ },
  setUser(id) { /* postMessage 到 iframe,通知更新身份(配合 v2) */ },
};
```

设计要点:

- **iframe 惰性创建**:用户首次点击才加载,首屏零开销。
- **z-index 极高值**(`2147483000` 之类),避免被平台元素遮挡。
- **响应式**:面板默认 `width:400px; height:640px`;窗口宽度 ≤ 500px 时改为占满全屏。
- **样式隔离**:所有 class 用 `momo-xxx` 前缀;CSS 用 `all: initial` 重置容器避免继承平台样式(可选)。

### 3.3 后端

**不改动。** `CORSMiddleware(allow_origins=["*"])` 已允许任意源,`ChatIn.user_id` 与 `/sessions?user_id=...` 已支持外部传入。`web/embed.js` 由 `app.mount("/", StaticFiles(...))`(`app/main.py:61`)自动作为静态资源对外提供,无需额外路由。

## 5. 验证步骤

按从小到大三步走:

### 5.1 嵌入模式自适应自测(无需平台)

```bash
docker compose up -d
uvicorn app.main:app --reload --port 8000
```

浏览器打开 `http://172.25.2.135:8000/?embedded=1&user_id=test-platform-user-1`:

- F12 → Network → 看 `/chat` 请求体里 `user_id == "test-platform-user-1"`,**不是** localStorage 的随机 UUID。
- 看 `/sessions?user_id=test-platform-user-1` 返回正确;切换 user_id query 后会话列表确实不同。
- 拖窗口到 420px 宽,确认 header / drawer / 气泡布局不破。

### 5.2 embed.js 在裸 HTML 里自测

写个 `test-host.html`:

```html
<body>
  <script src="http://172.25.2.135:8000/embed.js"></script>
  <script>MomoAgent.init({origin:'http://172.25.2.135:8000', userId:'demo'})</script>
</body>
```

双击打开:右下角悬浮按钮 → 点击展开 → iframe 加载 mom_agent → 发条消息能流式回复 → 关掉再开,会话状态保留。

### 5.3 真机接入 workbench

把 `<script>` 段粘到 `http://172.25.2.135:8080/workbench` 的模板尾部,`userId` 用平台模板变量渲染当前登录用户 ID。刷新后重复 5.2,并:

- 独立访问 `http://172.25.2.135:8000/?embedded=1&user_id=<同一平台用户ID>`,「会话」列表应包含刚才在 workbench 浮窗里产生的会话 —— 证明 user_id 串通。

## 6. 后续可选增强(v2)

- **postMessage 双向通讯**:平台切换用户时实时通知 iframe 更新身份,无需刷新。
- **主题适配**:`init({ theme: 'dark' })`,iframe 内根据 query 切换深色配色。
- **未读消息提示**:悬浮按钮上叠加红点(需要后端额外的"未读条数"端点,或由平台业务侧传入)。
- **CSP 与 CORS 收紧**:见第 4 节。
